logging best practices for log
1. log meaning full business events(order 101 created)
2. use appropriate log levels (info for normal flow,warn for recoverable issues,error for failiurs etc...)
3. use placeholders for unique id wise tracker we can  use logger.info
4. always include exception while login error
    logger.error("payment failed",e);
5. include request id's in distributed system to trace a request across services
6. avoid logging password , credit card number , authentication tokens or other sensitive data
7. avoid accessive debu logging in production because it can impact performace and increase storage cast

production troubleshooting workflow
1. identify the effected requested using a request Id, order id or customer id
2. filter loggs by that id
3. look for error and warn entries around the failiure time
4. review the stack trace to identify source of exception 
5. corelated loggs accross dependent services if the application is distributed
6. verify matrix (CPU usage ,memory ,database latency) to see if infrastructure was the resion for this issuie 
7. implement the the fix ,deployee it monitor new logs to confirm whether the problem is solved or not
