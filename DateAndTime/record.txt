limitation or records
1. can not extends another class
2. they implicitly final they can not be inharited
3. instance fields are fixed based on record component so we can not add extra instance fields
4. suitable for immutable for data carriers not with entities with changing data
   use records for 
   1. API require or response DTO's
   2. read only configuration object
   3. event payloads (kafka)
   4. value objects 
   5. data base projection result
   6. avoid records for JPA entities
   7. classes with significant business login
   8. object whos state changes over time
